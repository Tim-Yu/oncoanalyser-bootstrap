# Oncoanalyser GitHub packaging plan

This folder contains the bootstrap code for a GitHub-first distribution plan.

## Recommended layout

- `oncoanalyser-bootstrap` repo
  - `README.md`
  - `plan.md`
  - `scripts/package_bundle.py`
  - `scripts/restore_bundle.py`
  - `repos.example.json`
  - `manifest.json` generated by the packer
- `oncoanalyser-runtime` repo
  - small runtime artifacts and manifests
- `oncoanalyser-singularity` repo
  - Singularity cache shards
- `oncoanalyser-reference` repo
  - reference cache shards
- `oncoanalyser-reference-extracted` repo (optional)
  - extracted reference cache shards, if you want stub-mode validation to be reproducible

## Limits that drive the split

- GitHub repository files above 50 MiB trigger warnings.
- GitHub repository files above 100 MiB are blocked.
- GitHub repositories are healthiest under 1 GiB and should ideally stay under 5 GiB.
Given the current bundle size and raw-only download constraint, large payloads must be split into many parts and distributed across multiple repositories.

## Build

The packer creates tarballs, splits them into chunks, and writes a manifest that records sizes and SHA256 values.

Example:

```bash
cd /mnt/scratch/BYU/3.PackedupApps/3.1.Oncoanalyser
python3 scripts/package_bundle.py \
  --source /mnt/scratch/BYU/2.Oncoanalyser_packingup \
  --output build \
  --mode raw
```

Default raw mode uses 50 MiB parts to stay in a safe zone below GitHub's 100 MiB hard limit.

## Restore

The restore script downloads all parts listed in a manifest, verifies checksums, reconstructs the tarballs, and extracts them into the target directory.

Example:

```bash
python3 scripts/restore_bundle.py \
  --manifest build/manifest.json \
  --base-map repos.example.json \
  --output /path/to/controlled/env/oncoanalyser
```

## Notes

- Do not package `work/`, `output/`, or other generated run artifacts unless you explicitly want to archive a past run.
- Sample data is excluded from the distribution package.
- `ref_cache_extracted` is optional and skipped by default to reduce payload size.
- All downloads are expected via raw GitHub URLs.
